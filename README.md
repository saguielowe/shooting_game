# Shooting Game

一款使用 **C++17 与 Qt 6** 开发的 2D 横版射击对战游戏。

游戏支持本地双人对战、玩家对 AI、无尽模式与双 AI 观战，并实现了武器、装备、法术、词条、平台寻路、伤害结算和排行榜等完整机制。

当前版本：**v3.0**

## 项目简介

Shooting Game 最初是一个本地双人射击课程项目，随后逐步扩展为包含多种模式和复杂 AI 行为的完整游戏。

玩家可以在多层平台地图中移动、跳跃、拾取装备并使用不同武器作战。不同地图会改变移动与视野规则，武器、护甲、法术和词条则进一步影响战斗策略。

v3.0 在原有双人对战与 AI 模式的基础上，新增了法术系统、词条系统、无尽模式、双 AI 观战模式，并重构了 AI 与伤害计算逻辑。

## 游戏模式

### 本地双人模式

两名玩家使用同一键盘进行对战，支持：

- P1 / P2 独立键位
- 按键冲突检测
- BO3 / BO5 / BO7 多局赛制
- 武器、护甲、法术和地形交互

### 玩家对 AI

玩家可以与 AI 角色进行对战。AI 会根据地图、武器、角色状态和随机人格选择移动与攻击策略。

### 无尽模式

连续与 AI 对抗：

- AI 伤害随轮次递增
- 每轮结束后从三个随机词条中选择一个
- 玩家死亡后展示统计面板
- 支持排行榜记录

### 双 AI 观战模式

对局中按下 `F + G + H`，可以将 P1 交给 AI 控制，进入双 AI 对战模式。

## 核心系统

### 武器系统

当前包含：

- 拳头
- 小刀
- 实心球
- 步枪
- 狙击枪

不同武器具有独立的攻击方式、弹药、耐久和弹道机制。

近战武器、投掷武器和枪械分别适用于不同距离与战斗场景。

### 装备与护甲

- 轻甲可抵御部分近战伤害
- 防弹衣可吸收枪械伤害
- 装备拥有独立耐久
- 耐久耗尽后自动失效
- 部分装备会影响角色属性与战斗策略

### 法术系统

v3.0 新增三种法术：

- **定身术**
- **隐身术**
- **铜头铁臂**

每种法术拥有独立的：

- 冷却时间
- 释放条件
- 持续效果
- 交互规则

法术之间存在克制关系，例如：

- 定身可以打破隐身
- 定身可以打破铜头铁臂
- 铜头铁臂可以触发弹反
- 隐身状态下可触发破影一击

### 词条系统

游戏包含 **57 种词条**：

- 13 种通用词条
- 44 种法术专属词条

词条使用独立乘区叠加，无尽模式中每轮结束后可以进行三选一。

词条能够影响：

- 基础攻击力
- 武器效果
- 法术强度
- 冷却时间
- 生存能力
- 特殊战斗机制

### 地形系统

地图包含多层平台和特殊地形：

- 普通平台
- 冰地
- 草地

不同地形会改变游戏规则：

- 冰地降低移动控制能力
- 草地可以隐藏角色
- 多层平台影响移动、追击与武器使用策略

### 随机掉落物

地图会定时生成：

- 武器
- 护甲
- 医疗道具
- 属性强化道具

AI 与玩家都可以主动拾取掉落物。

## AI 系统

v3.0 对 AI 进行了较大重构。

### 分层有限状态机

AI 使用分层有限状态机组织行为，包含 8 个主要状态，并为高优先级情况设置特殊分支。

AI 会处理：

- 移动与追击
- 躲避与拉开距离
- 攻击
- 拾取物品
- 切换武器
- 释放法术
- 卡住后的脱困
- 特殊状态响应

### 随机人格

每局 AI 会获得不同人格，使行为不完全固定。

当前人格包括：

- **背水一战**
- **游走机动**

不同人格会影响 AI 的进攻倾向、移动方式与风险偏好。

### 平台寻路

AI 使用平台级 BFS 寻路，在多层地图中选择移动路径。

同时实现了卡住检测与脱困机制，避免 AI 长时间停留在无效位置。

### 武器决策

AI 可以：

- 根据当前状态切换武器
- 选择更适合距离和场景的武器
- 主动拾取掉落物
- 根据弹药和耐久调整策略

### 法术策略

AI 支持多种法术选择策略：

- `CONFIG_PICK`
- `COUNTER_PICK`
- `RAN`

这些策略分别用于配置选择、克制选择和随机选择。

## 伤害系统

v3.0 重构了伤害计算流程，使用多个独立乘区：

```text
基础攻击力
  × 武器倍率
  × 武器专精
  × 动能计算
  × 定身增伤
  × 护甲减伤
````

此外还包含：

* 基于伤害等级的硬直查表
* 铜头铁臂弹反
* 隐身破影一击
* 法术增伤与减伤
* 护甲耐久结算
* 不同武器的独立伤害机制

## 构建与运行

### 直接运行 Release

1. 在 GitHub Releases 页面下载最新版本
2. 解压 `shooting_game_release.zip`
3. 运行可执行文件

Windows Release 已包含所需 Qt 动态链接库，无需额外安装 Qt。

### 从源码构建

确保已安装：

* CMake
* Qt 6.5.3 或兼容版本
* 支持 C++17 的编译器

示例：

```bash
cmake -S . -B build
cmake --build build --config Release
```

不同平台和 Qt 安装方式可能需要额外指定 `CMAKE_PREFIX_PATH`：

```bash
cmake -S . -B build \
  -DCMAKE_PREFIX_PATH=/path/to/Qt/6.5.3/<compiler>
```

## 项目背景

本项目最初作为课程项目开发，之后持续进行功能扩展和架构重构。

项目开发过程中重点探索了：

* 如何使用面向对象方法组织复杂游戏系统
* 如何设计可扩展的武器、装备、法术和词条机制
* 如何在多层平台地图中实现 AI 寻路
* 如何在游戏效果、实现复杂度和可维护性之间进行取舍
* 如何使用 AI 编程工具辅助功能设计、代码实现和调试
