# Phase 1-5 最终完成记录（历史快照）

本文档记录的是“之前 Phase 1-5 最终完成时”的状态，不是最近一轮修复内容。

## 1. 目标范围（Phase 1-5）

本阶段聚焦 3 个问题：

1. 修复锁链甲在铜头反制后的“硬直补差值穿甲”问题。
2. 优化 AI 释放定身（FREEZE）时机，加入人格与武器收益判断。
3. 修复 gameai.h 的编译错误（包含 Qt 随机头文件与声明问题）。

## 2. 当时的最终改动

### 2.1 铜头反制相关（combatmanager.cpp）

- 处理了铜头反制与普通受击路径的分流。
- 在当时版本中，重点是确保“反制触发时攻击方进入硬直，且不走普通直伤路径”。

说明：
- 该阶段主要解决的是“反制后连段导致护甲被绕过”的结果问题。

### 2.2 硬直补伤与护甲结算（playercontroller.cpp）

- 在硬直中受击分支里，先应用护甲规则，再计算 overflow（新峰值相对旧峰值的差值）。
- 这样可避免“先做差值再减伤”带来的护甲穿透。

核心思路：
- `有效伤害 = 护甲结算后的伤害`
- `overflow = 有效伤害 - 旧峰值`
- 仅当 `overflow > 0` 才追加扣血与硬直峰值更新。

### 2.3 AI 定身时机（gameai.cpp）

- 释放窗口约束：距离与高度差都在可收益区间才考虑施放。
- 空中近战目标（拳/刀）时，AI 会降低或放弃该次定身。
- 人格差异：
  - `TRICKSTER` 更倾向冷却好就尽快放。
  - 其他人格更偏向“等高收益窗口”（如远程武器在手、伤害词条更优）。

### 2.4 编译修复（gameai.h）

- 增加了 `QRandomGenerator` 相关头文件，消除未声明标识符报错。
- 修复了声明结构问题，保证头文件可通过编译。

## 3. Phase 1-5 完成时的验证结果

当时已完成的验证结论：

1. 关键修改文件通过静态错误检查（相关逻辑文件无新增语法错误）。
2. 目标问题链路可解释：
   - 铜头反制触发 -> 攻击方硬直。
   - 硬直期间再次受击 -> 护甲先结算 -> overflow 再结算。
3. AI 定身行为从“固定偏随机”升级为“窗口 + 人格 + 武器收益”的组合决策。
